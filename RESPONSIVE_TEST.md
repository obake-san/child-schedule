## 🧪 テスト手順

### 1. デスクトップでテスト
```bash
# 開発サーバー起動
npm run dev
# ブラウザで http://localhost:5175 を開く
```

### 2. レスポンシブテスト（Chrome/Edge）
1. **DevTools を開く** → `F12`
2. **デバイスツールバーの切り替え** → `Ctrl+Shift+M`
3. **デバイス選択**:
   - iPhone 12 (390×844)
   - iPad (768×1024)
   - Pixel 5 (393×851)

### 3. ブレークポイントテスト
| デバイス | 幅 | テスト内容 |
|---------|------|-----------|
| モバイル縦 | 375px | 1列、タップしやすいボタン |
| モバイル横 | 667px（横向き） | スクロール対応 |
| タブレット | 768px | 2列グリッド |
| デスクトップ | 1024px+ | 3列グリッド |

### 4. UI 確認チェックリスト

#### モバイル (< 640px)
- [ ] ボタンが縦積みに配置されている
- [ ] フォント読みやすい（16px以上）
- [ ] モーダル幅が 95% になっている
- [ ] カレンダーがスクロール対応
- [ ] ヘッダーがコンパクト表示

#### タブレット (640-1023px)
- [ ] 児童一覧が 2列グリッド
- [ ] スケジュール一覧が 2列グリッド
- [ ] ボタンが見やすい（中程度）
- [ ] モーダル幅が 90%

#### デスクトップ (≥ 1024px)
- [ ] 児童一覧が 3列グリッド
- [ ] スケジュール一覧が 3列グリッド
- [ ] 標準的なレイアウト
- [ ] モーダルが中央表示

### 5. タッチデバイステスト
実際のスマートフォン/タブレットで以下をテスト：
```
テスト項目：
✓ ボタンのタップレスポンス
✓ モーダルのスワイプ閉じ
✓ スクロールのスムーズさ
✓ 入力フォームの操作性
✓ ピンチズーム対応
```

## 🔍 ブラウザDevToolsでチェック

### 計算済みスタイルの確認
1. モバイルレイアウトで児童カードを選択
2. DevTools > Elements > Computed タブ
3. `grid-template-columns` が `1fr` になっていることを確認
4. ウィンドウをリサイズして `1fr 1fr` (タブレット)、`1fr 1fr 1fr` (デスクトップ) に変わることを確認

### メディアクエリの確認
```javascript
// Console で実行
window.matchMedia('(max-width: 639px)').matches  // true = モバイル
window.matchMedia('(min-width: 640px) and (max-width: 1023px)').matches  // true = タブレット
window.matchMedia('(min-width: 1024px)').matches  // true = デスクトップ
```

## 📞 トラブルシューティング

| 問題 | 原因 | 解決策 |
|------|------|--------|
| メディアクエリが効かない | キャッシュ問題 | `Ctrl+Shift+Delete` でキャッシュクリア |
| DevTools で表示が違う | ビューポート設定 | `Ctrl+Shift+M` で再度切り替え |
| タッチスクロール遅い | パフォーマンス問題 | `will-change: transform` を追加 |
